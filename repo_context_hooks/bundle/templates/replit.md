# replit.md

## Project Context

- Start by reading `README.md`, `specs/README.md`, and `AGENTS.md`.
- Treat `README.md` as the user-facing source of truth for product intent and contribution expectations.
- Treat `specs/README.md` as the engineering continuity source for constraints, decisions, failures, and next work.

## Working Style

- Preserve useful continuity back into checked-in repo files instead of chat-only state.
- Before major changes, review the current repo structure and align with existing patterns.
- Keep handoff notes concise, concrete, and grounded in files that future sessions can inspect.

## Quality Guardrails

- Prefer small, reviewable changes over broad rewrites.
- Do not invent lifecycle hooks or automation that Replit does not expose.
- If the work is interrupted, leave the next session a clearer repo state than the one you inherited.
